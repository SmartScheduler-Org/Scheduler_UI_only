Our Journey
How SmartScheduler was built
💡
The Problem
The manual timetable nightmare
Our team witnessed college coordinators spending 2–3 weeks every semester building timetables in Excel, only to find teacher clashes and room conflicts on the first day. There had to be a better way.
🧬
The Algorithm
Building the genetic algorithm engine
We modeled timetabling as a constraint satisfaction problem and applied a genetic algorithm with custom fitness functions — weighing teacher workload, room capacity, lab sessions, and section uniqueness simultaneously.
🚀
The Product
Wrapping the engine in a usable UI
We built a Django web app that lets any administrator — with zero technical knowledge — input departments, courses, rooms, and faculty preferences, then generate a fully conflict-free timetable with one click.
🎯
Today
Teacher preferences, PDF exports & more
SmartScheduler now supports teacher preference collection via email, drag-and-drop manual adjustments, substitute scheduling, PDF exports, and role-based access for admins, teachers, and deans.
